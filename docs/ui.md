# 主界面设计

织册的用户界面是一个即来即走的单页工作台，不是资源库，也不是后台系统。

## 页面结构

```text
织册
把教材网页整理成可保存的 PDF

[ 教材链接输入框                              ]
[ 匿名限流提示 ]                      [生成 PDF]

当前任务
数学七年级上册
██████████░░░░  86 / 205 页
正在整理第 86 / 205 页
[下载 PDF] [复制下载链接] [重新开始]

最近下载
数学七年级上册  205 页  今天 14:20  [下载] [删除]
```

## 中心输入区

- 输入框是第一屏的主动作。
- 支持手动粘贴教材链接。
- 提供“粘贴并生成”按钮，用户点击后读取剪贴板并提交。
- 链接错误时显示“请粘贴智慧教育平台教材详情页链接”，不展示技术错误。
- 微信或手机浏览器中显示轻提示，引导用户用系统浏览器打开，避免大文件下载失败。

## 当前任务卡

任务提交后，在输入区下方展开任务卡。任务卡显示：

- 教材标题
- 页数
- 当前模式：云端优先或本机生成
- 状态 badge
- 进度条
- 面向老师的状态文案

状态文案：

- 正在读取教材信息
- 正在整理第 N / M 页
- 正在保存 PDF
- 已完成，可以下载
- 已完成，已开始下载
- 云端生成失败，已切换到本机生成
- 这个资源没有公开预览，无法生成 PDF

## 本地最近下载

- 最近下载只存在浏览器本地。
- 用户不需要登录。
- 每条历史记录显示教材名、页数、生成时间、下载按钮。
- 支持删除单条或清空全部。

## 移动端

移动端采用纵向布局：

```text
标题
输入框
匿名限流提示
生成按钮
当前任务
最近下载
```

主按钮全宽，历史项变成紧凑列表。
